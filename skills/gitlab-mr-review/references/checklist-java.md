# Java Review Checklist

## Code Consistency

- [ ] **命名规范**:
  - 类名: PascalCase
  - 方法名: camelCase
  - 常量: UPPER_SNAKE_CASE
  - 包名: 全小写
- [ ] **方法长度**: 不超过50行，过长应拆分
- [ ] **类长度**: 不超过500行，考虑拆分为多个类
- [ ] **参数数量**: 方法参数不超过5个，使用Builder模式或参数对象
- [ ] **魔法数字**: 使用具名常量
- [ ] **字符串常量**: 重复字符串应定义为常量

## Style Consistency

- [ ] **大括号**: 左大括号不换行（K&R风格）
- [ ] **缩进**: 使用4个空格
- [ ] **空格**: 关键字后、运算符前后有空格
- [ ] **行长度**: 每行不超过120字符
- [ ] **导入**: 避免通配符导入（import java.util.*）

## Code Duplication

- [ ] **DRY原则**: 相同逻辑提取为方法或工具类
- [ ] **继承 vs 组合**: 优先使用组合
- [ ] **工具类**: 通用逻辑放入util包
- [ ] **模板方法**: 相似算法使用模板方法模式

## Security

- [ ] **SQL注入**: 使用PreparedStatement，禁止字符串拼接SQL
- [ ] **XSS防护**: 输出前转义用户输入
- [ ] **敏感信息**: 不在日志中输出密码、token
- [ ] **文件操作**: 验证文件路径，防止目录遍历
- [ ] **反序列化**: 不信任的数据反序列化前验证
- [ ] **加密算法**: 使用强加密算法（AES-256, SHA-256）
- [ ] **依赖安全**: 检查Maven依赖已知漏洞

## Logic & Error Handling

- [ ] **空值检查**: 方法参数使用Objects.requireNonNull，避免NPE
- [ ] **Optional使用**: 返回值可能为空时使用Optional
- [ ] **异常处理**: 不捕获异常后什么都不做，至少记录日志
- [ ] **具体异常**: 捕获具体异常类型，不盲目捕获Exception
- [ ] **资源关闭**: 使用try-with-resources自动关闭IO/连接资源
- [ ] **并发安全**: 多线程环境使用线程安全集合，锁粒度合理
- [ ] **边界检查**: 数组/集合操作前检查索引，避免越界
- [ ] **分支完备性**: if/else覆盖所有分支，switch有default
- [ ] **循环终止**: 确保循环有明确的终止条件，避免死循环
- [ ] **集合判空**: 对List/Map/String等操作前先做null/空判断
- [ ] **数值安全**: 关注除零、整数溢出、浮点精度、BigDecimal构造方式
- [ ] **状态一致性**: 异常发生后对象仍保持合理状态，不留下半完成修改
- [ ] **幂等性**: 重复调用是否安全（如重复提交、重试场景）
- [ ] **死锁风险**: 避免嵌套锁、锁顺序不一致导致的死锁
- [ ] **字符串比较**: 使用equals/compareTo，避免用==比较字符串内容
- [ ] **时区/日期**: 日期计算考虑时区，避免使用已废弃的Date API
- [ ] ** equals/hashCode**: 重写equals时必须同时重写hashCode

## Performance

- [ ] **String拼接**: 循环中使用StringBuilder
- [ ] **集合初始化**: 预估大小初始化集合
- [ ] **IO操作**: 使用BufferedReader/Writer
- [ ] **懒加载**: 复杂对象考虑懒加载
- [ ] **数据库连接**: 使用连接池

## Testing

- [ ] **JUnit**: 使用JUnit 5
- [ ] **测试命名**: 测试方法名描述行为
- [ ] **边界测试**: 测试null、空集合、边界值
- [ ] **Mock**: 使用Mockito模拟依赖
- [ ] **覆盖率**: 核心业务逻辑覆盖率达到80%

## Documentation

- [ ] **JavaDoc**: 公共API有JavaDoc注释
- [ ] **复杂逻辑**: 复杂算法有注释说明
- [ ] **TODO标记**: TODO包含issue编号和责任人
