# JavaScript/TypeScript Review Checklist (AI Assisted)

> **定位说明**：本 checklist 用于补充 ESLint / Prettier / SonarQube 等静态工具难以发现的问题。对于分号、引号、缩进、行长度、简单空格等纯风格问题，**不单独提出**，除非它们严重影响了代码可读性或设计意图。

## Architecture & Design

- [ ] **单一职责**：函数/组件/类是否承担了过多职责，是否存在 Giant Component / God Function
- [ ] **抽象层次**：公共库/工具函数的接口是否稳定，业务代码是否耦合了底层实现细节
- [ ] **数据流方向**：状态提升或全局状态管理是否合理，是否存在逆向数据流（子组件直接修改父组件/全局状态）
- [ ] **副作用控制**：副作用（DOM 操作、网络请求、定时器、订阅）是否集中在合适的边界（如 React useEffect、服务层），而不是散落在纯函数中
- [ ] **配置 vs 硬编码**：业务规则、阈值、URL、枚举值是否应抽为配置或常量
- [ ] **扩展性**：新增同类功能时是否只需要新增少量代码（如新增路由、新增策略、新增表单字段），而不需要修改既有核心逻辑
- [ ] **类型抽象**：TypeScript 类型定义是否足够表达领域模型，是否存在过度使用 `any` / `unknown` 而未做 narrowing

## Logic & Correctness

- [ ] **分支完备性**：if/else/switch 是否覆盖了所有业务场景，是否存在隐式遗漏（如缺少 default/else）
- [ ] **异步时序**：
  - async/await 与 Promise 混用是否导致意外时序
  - 竞态条件：先发出的请求后返回，是否覆盖了最新状态
  - 并行请求的错误处理是否完整（Promise.all 一个失败是否导致全部丢失）
- [ ] **闭包陷阱**：异步回调、useEffect、事件监听中是否引用了过时的闭包变量
- [ ] **事件与订阅**：事件监听、WebSocket、RxJS 订阅是否在组件卸载/页面关闭时正确清理，避免内存泄漏或重复绑定
- [ ] **边界条件**：数组/对象操作前是否检查空值、长度、key 存在性，避免 undefined 导致的级联错误
- [ ] **短路求值误用**：`a || b` / `a && b` 是否用于非布尔场景并引入了副作用或意外值（如 `0 || defaultValue`）
- [ ] **深拷贝与浅拷贝**：修改对象/数组时是否意外 mutates 了共享引用，导致不可预期的状态变更
- [ ] **浮点精度**：涉及金钱的数值计算是否避免了直接浮点运算
- [ ] **幂等性**：重复提交、重试、回调通知等场景是否安全

## Security (Human Judgment Required)

- [ ] **XSS 风险**：用户输入是否直接插入了 DOM（innerHTML、dangerouslySetInnerHTML、v-html）或 URL
- [ ] **URL 跳转**：外部链接跳转、iframe src、window.open 是否经过白名单校验
- [ ] **eval / new Function**：是否存在执行动态代码的风险入口
- [ ] **敏感信息泄露**：前端代码、日志、错误上报中是否硬编码或泄漏了密钥、token、个人隐私数据
- [ ] **依赖安全**：新增 npm 包是否有已知 CVE（尤其是解析类、正则类、网络请求类库）

## Data Flow & State

- [ ] **状态一致性**：多个关联状态是否在更新时保持同步（如分页列表的 currentPage 与 data 不一致）
- [ ] **缓存与同步**：前端缓存（SWR、React Query、Redux、Pinia）与服务器状态是否保持一致，失效策略是否合理
- [ ] **乐观更新**：乐观更新失败后的回滚策略是否完整
- [ ] **表单校验**：复杂表单的多字段联动校验、异步校验、服务端校验是否覆盖完整

## Maintainability & Extensibility

- [ ] **魔法数字/字符串**：业务含义不明确的裸数字/字符串是否应提取为具名常量或枚举
- [ ] **重复逻辑**：相同业务逻辑是否在多处硬编码，应提取为 hooks / utils / services
- [ ] **注释质量**：复杂算法、非常规实现、业务规则是否有足够注释说明“为什么”
- [ ] **兼容性风险**：使用了实验性 API 或浏览器专属 API 时，是否有降级方案
