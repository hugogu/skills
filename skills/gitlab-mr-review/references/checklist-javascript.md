# JavaScript/TypeScript Review Checklist

## Code Consistency

- [ ] **命名规范**: 变量使用camelCase，常量使用UPPER_SNAKE_CASE，类使用PascalCase
- [ ] **函数长度**: 函数不超过50行，超过应拆分
- [ ] **文件长度**: 单个文件不超过500行
- [ ] **魔法数字**: 使用具名常量代替魔法数字
- [ ] **默认导出**: 避免使用默认导出（`export default`），优先使用命名导出
- [ ] **箭头函数**: 在适用场景使用箭头函数
- [ ] **解构赋值**: 优先使用解构获取对象属性

## Style Consistency

- [ ] **分号**: 一致地使用或省略分号
- [ ] **引号**: 统一使用单引号或双引号
- [ ] **缩进**: 使用2或4个空格，保持一致
- [ ] **尾随逗号**: 多行对象/数组的最后一个元素使用尾随逗号
- [ ] **空格**: 运算符前后、逗号后有空格

## Code Duplication

- [ ] **DRY原则**: 相同代码块（>3行）应提取为函数
- [ ] **工具函数**: 通用逻辑应放入utils目录
- [ ] **组件复用**: UI组件应避免重复实现

## Security

- [ ] **XSS防护**: 用户输入插入DOM前必须转义
- [ ] **eval禁用**: 不使用eval()或new Function()
- [ ] **innerHTML**: 避免使用innerHTML，优先使用textContent
- [ ] **敏感信息**: 不在代码中硬密码钥、token
- [ ] **依赖安全**: 检查依赖包是否有已知漏洞
- [ ] **URL验证**: 外部URL跳转前验证域名
- [ ] **正则表达式**: 避免ReDoS风险的正则

## Logic & Error Handling

- [ ] **空值检查**: 使用可选链（?.）和空值合并（??）
- [ ] **类型检查**: TypeScript项目中避免使用any，确保类型 narrowing 正确
- [ ] **错误处理**: 异步操作使用try/catch或.catch()，错误有明确传递
- [ ] **边界条件**: 数组操作前检查长度，避免访问 undefined
- [ ] **Promise处理**: 避免unhandled promise rejection，async/await 有对应 try/catch
- [ ] **事件监听**: 及时移除不再使用的事件监听，避免内存泄漏
- [ ] **内存泄漏**: 注意闭包、定时器、订阅的清理
- [ ] **分支完备性**: if/else 覆盖所有分支，switch 有 default
- [ ] **短路求值误用**: 避免 `a || b` / `a && b` 用于非布尔场景的副作用陷阱
- [ ] **严格相等**: 使用 `===` / `!==`，避免类型隐式转换导致的意外结果
- [ ] **深拷贝与浅拷贝**: 修改对象/数组时避免意外副作用
- [ ] **浮点精度**: 涉及金钱的数值计算避免直接用浮点数运算
- [ ] **竞态条件**: 异步请求返回顺序是否与UI状态一致，避免先发出的请求后返回覆盖最新状态
- [ ] **三元嵌套**: 三元表达式嵌套过深时改用 if/else 或策略表提高可读性

## Performance

- [ ] **循环优化**: 避免在循环中进行DOM操作
- [ ] **防抖节流**: 高频事件使用debounce/throttle
- [ ] **懒加载**: 大组件/模块考虑懒加载
- [ ] **避免重复计算**: 复杂计算使用useMemo或缓存

## Testing

- [ ] **测试覆盖**: 核心逻辑有单元测试
- [ ] **边界测试**: 测试边界条件和错误场景
- [ ] **Mock数据**: 外部依赖使用mock
