# Project Review Checklist

## Custom Rules

Add your project-specific review rules here:

- [ ] 所有API调用必须有超时设置（timeout参数）
- [ ] 日志必须使用结构化格式（JSON）
- [ ] 数据库查询必须使用参数化查询
- [ ] 敏感操作必须记录审计日志
- [ ] 新API必须有对应的单元测试

## Disabled Rules

禁用以下内置规则（如果需要）：

- 禁用: "no_console_in_prod"（本项目允许console.log）
- 禁用: "function_length"（本项目允许长函数）

## Severity Overrides

调整规则的严重程度：

- 将 "missing_docstring" 设为 warning（而非 info）
- 将 "unused_import" 设为 warning

## Language-Specific Rules

### JavaScript/TypeScript
- [ ] 所有异步函数必须使用try/catch处理错误
- [ ] React组件必须有PropTypes或TypeScript类型定义

### Java
- [ ] 所有服务类必须使用Spring的@Transactional注解
- [ ] API响应必须使用统一的Response包装类

### Python
- [ ] 所有函数必须有类型注解
- [ ] 使用pydantic进行数据验证
